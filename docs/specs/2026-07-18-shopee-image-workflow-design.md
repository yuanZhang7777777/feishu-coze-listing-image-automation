# Shopee 1:1 Listing 营销图工作流设计

日期：2026-07-18  
状态：V2 通用工作流重建基线；旧单图流程仅保留作对照

本文描述重新搭建的 Shopee 1:1 Listing 图片 V2 通用工作流。旧工作流不继续局部修补、不连接飞书，只保留作效果对照；飞书自动化和云服务器中转服务已经接入。2026-07-19 首轮真实 E2E 暴露的 `生成详情图` 默认 3 分钟超时已修正；第二轮连续运行 22 分 34 秒并成功生成 7 张 Listing 图片，证明长时生图正常。中转的大写 `Output` 兼容已部署，同一执行记录已回写 1 张标准商品图和 7 张 Listing 图片到飞书，完成态幂等验收通过。ID 12 经明确授权只重跑一次，新执行运行 19 分 51 秒后完成；标准图空字段最终定位为 `拆分最终结果` 代码返回键和短输出定义不一致，固定字段契约现已稳定。2026-07-20 已发布 `v0.0.12`：只有 `reuse` 直接复用原图，其他模式使用 `标准商品图生成` 的输出；实验抠图节点保持断开。ID 12 已无付费补回原商品图，最终为 1 张标准商品图和 7 张 Listing 图片。`zhandian` 传递正确，但 TH/VN 等站点仍出现英语消费者文案；完整本地化提示词已增加目标语言硬映射和输出前语言闸门。飞书多图上传、竞品分析、标准商品图、中转服务和结果回写的目标架构，以 [飞书多维表格 Listing 自动出图系统设计](2026-07-18-feishu-listing-image-automation-design.md) 为准。

## 1. 目标

生成 6–8 张、1:1 方形 Shopee Listing 营销图。

本工作流统一生成 1:1 Listing 图片；第一张承担商品主图/首图作用，但不强制纯白底，其余图片负责种草、卖点说明、使用场景、细节展示和信任收尾。

## 2. V2 完整架构

开始（产品名称、多张自家产品图、商品事实、可选竞品图、平台、站点）  
→ 自家多图商品理解  
→ 主商品置信度判断  
→ 条件分支：待补充资料 / 继续处理  
→ 标准商品图策略判断  
→ 条件分支：复用干净图 / 普通抠图 / 语义提取  
→ 条件分支：有竞品图时执行竞品策略分析  
→ Shopee 详情页设计  
→ JSON 数组解析（仅设计输出仍为 String 时需要）  
→ 批处理  
→ 本地化与生图 Prompt 编译  
→ 生图插件（绑定标准商品图作为商品参考）  
→ 汇总  
→ 结构化结束

V2 是一条新工作流，不在旧单图工作流上继续叠加节点。付费生图节点之前的商品理解、置信度判断、标准商品图、竞品隔离和分屏设计必须先单独试运行通过。

### 详情页设计 AI

输出 6–8 条完整的单图中文设计字符串。每条包含本张消费者文案源、画面目标、商品呈现、人物与场景、构图、摄影风格和本图防错；下游“本地化与生图 Prompt 编译”再把当前一条设计转换成最终可生图 Prompt。

它不输出空泛分析，也不把“引流屏、功能转化屏”等内部标签写进设计字符串。

### 本地化与生图 Prompt 编译 AI

它不是逐字翻译器。它接收当前分屏设计、商品身份锁和站点，依据 zhandian 将标题、副标题和短标注改写为目标市场自然、通顺、简洁的电商文案，并把设计编译成可直接投喂生图插件的单条完整 Prompt。最终 Prompt 的画面描述、摄影指令、商品约束和禁止事项统一使用英文；只有实际需要显示在图片中的消费者文案，以及商品本身真实存在的品牌/型号，保留目标站点语言。商品外观、结构数量、人物、场景、构图和摄影风格不得在编译中丢失。

### 生图插件

- prompt 绑定语言本地化节点完整输出。
- 参考图字段绑定真实商品图。
- 画幅固定 1:1。
- 每轮生成一张。
- `asyn=true`（同步等待）；节点超时时间 10 分钟；自动重试 0。

旧“图片格式转换”节点不进入 V2。参考图处理由“标准商品图策略判断与执行”按 reuse、cutout、semantic_extract 动态完成；设计 JSON 只交给“JSON 数组解析”代码节点，二者不得混用。

Coze `v0.0.12` 的生产连线为两路：`reuse → 选择标准图源（URL）→ 复用标准图 URL`；`cutout/semantic_extract → 选择标准图源（URL）→ 组装标准商品图 Prompt → 标准商品图生成`，两路再进入标准图结果收口。用户后加的 `按索引抠图` 实验批处理曾只保留一个商品组件，当前保持断开；不得把其 `Array<Image>` 输出绑定到要求 `Array<String>` 的标准图生成参考图字段。未来若重新启用非生成式 cutout，需要另行完成完整主体、索引和 URL 契约验证。

当前生图插件输出的 `data.url` 是 `String`，Coze AI 节点的视觉理解输入要求真实 `Image`。第一版跳过标准图和详情图的 AI 视觉一致性检查，不做 String 到 Image 的强制绑定；联调阶段人工抽查图片 URL。只有后续增加可靠的 URL→Image 转换或替换插件/视觉 API 后才恢复自动质检。

## 4. 输入

- product_name：运营填写的明确商品名称或具体款式，用于多物体、多款式和包装混拍中的主商品定位。
- product_images：1–10 张自家商品图片，可混合白底图、场景图、细节图、人物使用图、包装盒、说明书和配件；系统必须动态判断各图用途。
- points：真实商品品类、卖点、规格、材质、包装清单、使用说明和限制；关键数量、排列或不可改变部件应明确写成“硬性结构事实”。
- competitor_images：可选竞品图片，只用于策略分析。
- platform：第一版固定 Shopee。
- zhandian：SG/MY/TH/VN/PH/ID/TW/BR。

语言固定映射：SG→英语、MY→马来西亚马来语、TH→泰语、VN→越南语、PH→英语、ID→印度尼西亚语、TW→繁体中文、BR→巴西葡萄牙语。

## 5. 通用设计原则

- 不预设任何具体商品属性；一切展示内容都从当前输入动态选择。
- 先判断商品的展示需求，再从可用信息中选择 6–8 个模块。
- 前六个通用任务为：视觉吸引、核心收益、事实证据、使用理解、细节品质、生活场景。
- 只有 points 存在对应事实时，才增加规格适配、包装内容、选购提醒等模块。
- 商品与人体、穿戴、携带、手持或操作有关时，安排 1–3 张人物或手部场景；不需要人物时使用真实尺度或使用环境。
- 所有营销声明只能来自 points；参考图只用于锁定商品外观和明显可见结构。
- 画面必须像成熟跨境电商详情页，不做白底商品目录、空洞艺术海报或无意义装饰图。

## 6. 八站视觉适配

站点只控制环境、光线、家居/工作空间和人物整体气质，不强制刻板五官、肤色或文化符号。

- SG：现代都市公寓或办公环境，浅灰、米白、干净利落。
- MY：明亮现代热带居家环境，暖灰、米白、自然光。
- TH：温暖明亮的现代居家环境，暖白和柔和低饱和点缀色。
- VN：紧凑现代居家或城市生活环境，暖中性色、自然光。
- PH：明亮通透的现代热带居家环境，白色、浅青和暖米色。
- ID：温暖实用的现代居家环境，暖灰、浅木和柔和点缀色。
- TW：现代小户型或城市生活环境，莫兰迪灰、奶白和浅木色。
- BR：明亮自然的现代居家或户外生活环境，暖中性色和克制活力色。

不加入国旗、著名地标、民族服饰、宗教符号、招牌或与商品无关的地域道具。

## 7. 事实边界

- points 是营销文案、数字、功能、材料、性能、适配、认证和效果的唯一来源。
- 参考图只用于确认颜色、整体形态、明显可见部件和包装外观。
- 不得仅凭图片推断精确数量、材料名称、机械属性、容量、承重、耐用性、安全性、适配范围或体验收益。
- 精确部件数量未写入 points 时，禁止从图片自行计数，只能使用不带数字的中性描述。
- points 明确提供的关键数量、排列、相对位置和不可改变部件属于“商品身份硬约束”，必须原样带入每一条完整生图 Prompt，不能只在其中一张出现。
- points 没有提供的声明必须省略；不要在 Prompt 中列举具体禁词示例，以免污染其他品类。
- 可以设计真实使用场景，但场景本身不能变成未经证实的商品宣称。穿戴、手持、携带、接触身体、需要操作或需要人物尺度才能理解的商品，6–8 张中应有 2–3 张真人或手部使用场景，且前两张至少一张；人物不是详情图禁止项。
- 输入可包含同一商品家族的多个颜色、花纹、常规尺寸子 SKU 及其不同角度。系统先选择本次主 SKU，再综合其他子 SKU 的互补角度锁定共有结构；最终一套生成图始终保持主 SKU 的颜色、花纹、Logo、接口、部件、配件和关键数量，不得混成不存在的跨 SKU 外观。

## 8. 人物与场景

- 人物不是默认禁止项，也不是每张必需项。
- 商品需要穿戴、携带、手持、接触身体或展示操作时，应安排人物或手部，让消费者理解使用关系和尺度。
- 人物场景之间改变环境、景别、姿态、机位和服装。
- 商品始终是画面主角，人物不得遮挡关键结构。
- 人物应自然、成年、尊重目标市场，不使用夸张表情、刻板地域特征或不必要裸露。
- 对无需人物解释的商品，可以使用桌面、空间、收纳、安装位置、组合关系或尺度参照。

## 9. 视觉与可见文字

- 固定 1:1 方形画布，画面铺满画布。
- 禁止黑边、信箱边、嵌套画框、巨大空白、无意义岩石、商品悬浮和无理由重复商品。
- 真实商业产品摄影和专业电商精修质感，柔和自然光，材质清楚，色彩通透。
- 同套保持字体气质、圆角、图标、留白、色温和辅助色统一；相邻两张至少改变三项构图元素。
- 每张只安排一个主标题、一个可选副标题和最多三个短标注。
- 消费者可见文案必须放在中文书名号「」中，书名号之外不得出现需要翻译的消费者文案。
- 不得在任何最终 Prompt 中写“第几屏、引流屏、功能转化屏、场景体验屏、信任促单屏、模块、screen、module、layout”等内部标签。

## 10. 验收

1. 详情页设计输出为 6–8 项 JSON 字符串数组。
2. 每项是当前图片的完整具体设计，不是抽象分析；本地化编译节点每轮只接收一项并输出一条完整生图 Prompt。
3. 全部固定 1:1；第一张承担商品主图/首图作用，可根据品类和真实用途采用干净商品英雄图或真人使用场景。
4. 本地化编译节点依据当前单图设计生成目标站点母语文案与最终生图 Prompt；译文符合语法、拼写、地区词汇和移动端可读性，画面意图与商品身份锁保持不变。
   最终 Prompt 除目标站点可见文案和商品真实品牌/型号外，其他画面描述与控制指令均为英文。
5. 人物数量和使用关系根据商品动态决定。
6. 不出现内部标签、黑边、空洞海报或无意义商品重复。
7. 所有声明能追溯到 points，全部商品外观与参考图一致。
8. points 中的商品身份硬约束出现在每条生图 Prompt 中；未提供的精确数量没有被模型从图片推断。
9. 批处理输入有 6–8 项时全部执行完成；标准图和详情图任一付费插件任务都不得因默认 3 分钟阈值产生 `plugin timeout`，且失败时不得自动重复付费。

## 11. 旧 V1 详情页设计节点 Prompt（仅历史对照）

以下 Prompt 使用旧 `cp_tu` 单图输入并直接生成最终 Prompt，不得用于 V2。V2 当前节点配置和完整 Prompt 以 [Coze Shopee Listing 通用出图 V2 实施计划](../plans/2026-07-19-coze-shopee-v2-workflow-implementation.md) 为准。

~~~text
# 角色定位

你是深耕 Shopee 跨境电商详情页视觉转化的资深视觉策划师，精通 SG、MY、TH、VN、PH、ID、TW、BR 八个买家市场的移动端竖屏浏览习惯、本地生活空间、人物使用场景、商业摄影审美和详情页递进转化逻辑。

你的任务不是写分析报告，而是根据真实商品参考图、商品事实与站点，为同一商品一次性规划并输出 6–8 条可以直接交给生图模型的完整中文生图 Prompt。

你要把重点放在“这个市场应该怎么拍、这一张承担什么销售任务、商品怎样出现、人物怎样互动、文案放在哪里、整套怎样保持统一”。限制只用于守住商品真实性和平台合规底线。

# 输入变量

商品参考图：{{cp_tu}}
商品真实资料与卖点：{{points}}
Shopee 站点：{{zhandian}}

# 总体目标

把 {{points}} 中已确认的商品价值按消费者决策顺序整理成一套 6–8 张、1:1 方形 Shopee Listing 营销图。

整套图片应让消费者依次完成：

第一眼认出商品并产生兴趣；
理解最重要的购买收益；
看到支持卖点的商品事实；
看懂商品怎样使用或怎样融入生活；
确认外观、结构、做工或选购细节；
在本地化生活场景中形成代入并愿意继续下单。

这套工作流统一生成 1:1 Listing 图片。第一张要在缩略图状态下清楚识别商品，并根据品类选择干净商品英雄图或真实人物使用图；后续图片继续承担详情页营销任务。每张既要像可直接上架的成熟电商设计稿，也要保持真实商业摄影质感，不能成为只有商品和大面积空白的艺术海报。

# 八个站点的正向视觉映射
~~~

~~~text
保持原文的宣传强度。普通描述不能改成最高级、绝对化、医疗功效、安全保证或其他更强承诺。

如果中文原文直译后不自然，应在不改变事实的前提下进行母语化改写；如果原文已经是该站点语言，也要检查它是否符合 {{zhandian}} 的地区用词和自然表达，必要时进行本地化润色。

每组「」分别处理，但要检查主标题、副标题和短标注之间是否重复、冲突或语气不一致。不得为了变化而给同一事实换成不同术语。

# 处理边界

只改写中文书名号「」内部的消费者可见文字。

书名号外是已经完成的生图控制指令，必须保持原有内容、顺序、数字和结构，不重新设计画面，不增删人物、场景、商品、构图、画幅、摄影风格或商品约束。

保留每一组书名号及其原位置。书名号是工作流用于识别可见文案的边界标记，不属于需要显示在图片上的字符。

不要新增书名号，不把书名号外文字移入书名号，也不要把书名号内消费者文案移到外部。

# 母语质量检查

输出前逐组检查：

站点对应语言和地区变体是否正确；
语法、拼写、声调符号、重音符号和标点是否正确；
词序和搭配是否像当地人自然表达，而不是中文逐字翻译；
标题、副标题和短标注是否适合手机端快速阅读；
是否准确保留品牌、型号、数字、单位、限定条件和事实强度；
是否出现马来语与印尼语、简体与繁体、欧洲葡萄牙语与巴西葡萄牙语等地区混用；
是否新增了原文没有的功能、承诺、夸张词或促销信息；
书名号外内容是否完全没有改变。

发现不自然表达时，先在书名号内改成地道说法，再输出最终完整 Prompt。

# 输出格式

只输出本地化后的完整生图 Prompt。

不要输出 JSON、Markdown、分析、翻译说明、语言名称、修改记录或中外文对照。

不要使用额外引号包裹整条输出。
~~~
